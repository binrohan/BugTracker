About Bug_Tracker V 0.1

Bug tracker is an application for tracking error or bug in software. It is developed by Rohan Bin Khokon as a student project for job preparation in 2020.

Bug Tracker uses a ticketing system to record bugs or other issues in the software development process on a per-project basis. It implements user- and role-level security to ensure only authorized users can access tickets and projects.

The Bug_Tracker application was build using ASP.NET Core with SQL Server database. Development was completed with Visual Studio Code. Bug_Tracker's webpages built using Angular 10, Angular material theme and some other material components.

Roles: Bug_Tracker has 3 roles that users can be assigned to. A user can be in more than one role at one time. After registration to this application, user have to wait for admin approval by assigning user one of the three roles. Then new user can use the application

Below are brief descriptions of the permissions of each role:

Developer

  • View assigned tickets
  • Post comments on projects' ticket developer assigned
  • Change status(Working, Waiting, On-Hold) of tickets
  • Pass assigned tickets to manager for approval

Project Manager or Manager

  • Create new ticket for assigned project
  • Edit ticket for assigned project
  • Assign or remove users from project
  • Approve tickets that are resolved by developer
  • Send potiential resolved ticket to admin for mark as resolved
  • Assign developer to ticket

Admin

  • Assgin users to roles
  • Create project
  • Archive project
  • Assign user to project
  • Remove user from project
  • Access any project
  • Create new ticket
  • Edit any ticket
  • Assign or remove user from any ticket
  • Mark ticket as resolved
  • Post commeont in any ticket
  • Add new value to ticket Priority, Status, Category
